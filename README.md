# PM IDE & AI Tools — Learning Sandbox

Welcome. You've found your way to the session workspace.

This repository is your home base for today — a safe, low-stakes environment where you can try things, make mistakes, and build real confidence working in an IDE with AI tools alongside you. Nothing here will break. Everything is reversible. You're encouraged to experiment.

No coding experience needed. Everything in this repo is plain text and markdown. If you can write a Slack message, you can work here.

---

## What this is

This sandbox is built around a single fictional product feature: **AI Meeting Intelligence**, part of an internal knowledge platform called **Meridian** at a technology consultancy called **Luminary Group**.

You'll find a PRD, a decisions log, and a changelog — all for this same feature, all written as if you're a PM on the team. These are your working files throughout the day.

The exercises will walk you through how to use an IDE and AI tools to do things you already do — just faster, with more context, and without switching between six different tabs.

Throughout this session, you're working on a feature called **AI Meeting Intelligence** for **Meridian**, Luminary Group's internal knowledge and collaboration platform.

## How the session is structured

### Block 1 — Git Basics

**Exercise:** `exercises/01-git-basics.md`
**Working file:** `docs/prd-template.md`

Learn how to clone this repo, make a change, and open a pull request. Every command is written out exactly as you should type it, with a plain-English explanation of what just happened. By the end, you'll have made your first contribution to a shared repository.

---

### Block 2 — Working with GitHub Copilot

**Exercise:** `exercises/02-copilot-modes.md`
**Working files:** `docs/prd-template.md`, `docs/decisions-log.md`

Explore three modes of Copilot — Chat, Edits, and Agent — and understand when each one is the right tool for the job. You'll use Copilot to improve the PRD, fill in placeholders, and surface inconsistencies between documents.

---

### Block 3 — MCP Servers & Connected AI

**Exercise:** `exercises/03-mcp-setup.md`
**Working files:** All docs

Learn what MCP servers are and why they matter, then see how connecting Copilot to tools like Confluence, Figma, and Mixpanel changes what's possible. You'll work through three realistic PM scenarios showing what this looks like in practice.

---

## A few things to know before you start

- **You won't break anything.** This is a sandbox. Experiment freely.
- **Work at your own pace.** Each exercise is self-contained — you can pause, go back, or skip ahead.
- **The AI is a collaborator, not an oracle.** It's wrong sometimes. Read what it produces, edit it, make it yours.
- **Ask questions out loud.** If something doesn't make sense, that's useful signal for everyone in the room.
